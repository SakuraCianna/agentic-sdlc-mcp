# v1.7.0 Registry and Repository Policy Implementation Plan

> **Implementation status (2026-07-13):** Complete. npm v1.7.0 published, but Registry correctly rejected a lowercase namespace that did not match the case-sensitive GitHub login. The immutable correction is v1.7.1 with `io.github.SakuraCianna/agentic-sdlc-mcp`.

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish-ready MCP Registry metadata plus a strict, explainable `.agentic-sdlc.yml` policy foundation consumed consistently by the six v1.7 tools.

**Architecture:** Add a focused `src/policy/` module that owns schema validation, safe YAML parsing, path matching, source/digest metadata, and repository loading. Tool handlers receive one immutable policy load result so they never parse YAML independently. Registry metadata is validated by repository tests before release; Registry publishing remains release-triggered and runs only after npm exposes the exact version.

**Tech Stack:** TypeScript 6, Zod 4, yaml 2.9, Octokit 22, Vitest 4, MCP Registry `server.json` schema 2025-12-11, mcp-publisher v1.7.9.

**Files:**

- [ ] **Step 1: Write failing schema tests**

Cover minimal valid policy, full policy, unknown keys, unsupported schema version, duplicate rule IDs, unknown `riskRuleIds`, empty reviewer lists, invalid globs, forbidden capability keys, oversized input, duplicate YAML keys, deep nesting, aliases, and stable digest.

- [ ] **Step 2: Run the policy test and confirm RED**

Run: `npm test -- --run src/__tests__/policy/repository-policy.test.ts`

Expected: FAIL because the policy module does not exist.

- [ ] **Step 3: Implement strict types and schema**

```ts
export interface RepositoryPolicyLoadResult {
  policy: EffectiveRepositoryPolicy;
  found: boolean;
  degraded: boolean;
  policySources: PolicySource[];
  appliedRules: AppliedPolicyRule[];
  ignoredRules: IgnoredPolicyRule[];
  errors: string[];
  warnings: string[];
}
```

Use `.strict()` at every object level. Reject the complete repository policy on any validation error and return immutable v1.6-compatible defaults.

- [ ] **Step 4: Implement bounded YAML parsing and digest**

Cap source at 64 KiB, reject duplicate keys and parser errors, limit aliases, normalize validated JSON recursively, and hash with SHA-256. Never digest raw YAML formatting.

- [ ] **Step 5: Implement path and reviewer selectors**

Support `/`, `*`, `**`, and `?` with normalized forward slashes. Reject absolute paths, traversal, control characters, empty selectors, and patterns above the length cap. Reviewer rules match when any referenced risk rule or any path selector matches.

- [ ] **Step 6: Run focused tests**

Run: `npm test -- --run src/__tests__/policy/repository-policy.test.ts`

Expected: PASS.
